## Executive read

The two tracks are compatible, but they operate at different altitudes.

- **Narrative-quality pass** fixes the most visible failure mode: SF2 turns pressure into paperwork because prompts and setup contracts under-specify human consequence.
- **Domain-model hardening** fixes the underlying schema/retrieval contracts so the same drift does not reappear as soon as prompts change.

Keep both tracks, but do not implement them as two independent backlogs. Use the narrative-quality pass as the near-term behavior layer, and use the domain-model track as the contract hardening layer.
