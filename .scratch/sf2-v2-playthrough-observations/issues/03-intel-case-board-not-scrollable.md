# Intel / Case Board section is not scrollable

Category: bug
Status: needs-triage
Area: SF2 / V2 UI / right rail layout
Reporter: Martin
Observed: 2026-05-02

## Observation

During a space opera SF2 playthrough, the `Intel / Case Board` section in the V2 UI was not scrollable.

## Expected

When the Intel / Case Board content exceeds the available rail space, the player should be able to scroll through the section without losing access to clues, threads, or floating clues.

## Lightweight Code Note

No fix attempted. Initial inspection points at a possible desktop layout conflict in `components/sf2/play-shell.tsx`:

- The desktop rail itself uses `overflow-y-auto`.
- `IntelPanel` wraps the case board in `HudPanel` with `className="min-h-0 flex-1 overflow-hidden"`.
- The inner case-board content has `overflow-y-auto`, but also `max-h-[42vh] ... xl:max-h-none`.
- On `xl` desktop layouts, the inner scroll container loses its max-height, while the outer panel clips overflow, which can make the section feel non-scrollable or hide overflowing content.

## Triage Questions

- Is the desired behavior for the whole right rail to scroll as one column, or should the Intel / Case Board have its own internal scroll area?
- Should Locations and Present remain fixed while Intel scrolls, or should all three panels participate in one rail scroll?
- Does the bug reproduce only on desktop `xl` layout, or also in the mobile drawer?
