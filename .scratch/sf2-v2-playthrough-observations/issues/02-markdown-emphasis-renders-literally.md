# Markdown emphasis renders literally in V2 prose

Category: bug
Status: needs-triage
Area: SF2 / V2 UI / narrative rendering
Reporter: Martin
Observed: 2026-05-02

## Observation

During a space opera SF2 playthrough, narrator prose included Markdown-style emphasis, but the V2 UI rendered the raw markers instead of formatting them.

Example:

- Saw `*Ashkara*` in the UI instead of italicized `Ashkara`.

## Expected

Player-facing narrative text should not expose raw formatting syntax. Either Markdown emphasis should render properly, or the Narrator should be constrained/normalized to avoid Markdown markers in prose.

## Lightweight Code Note

No fix attempted. Initial inspection points at the V2 narrative renderer:

- `components/sf2/play-shell.tsx` renders both stored and live prose through `GMMessage`.
- `GMMessage` renders `{children}` directly inside a styled div with `whitespace-pre-wrap`.
- There does not appear to be a Markdown renderer in this V2 path, so Markdown syntax remains literal.

## Triage Questions

- Should SF2 V2 support Markdown rendering for narrator prose, or should narrator output be plain text only?
- If rendering Markdown, which subset should be supported: emphasis only, paragraphs, lists, inline code, headings?
- Should live streaming text and stored history use the same renderer to avoid formatting changing after the turn commits?
