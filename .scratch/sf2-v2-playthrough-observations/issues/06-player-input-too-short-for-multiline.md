# Player input should show at least two lines before scrolling

Category: enhancement
Status: needs-triage
Area: SF2 / V2 UI / player input ergonomics
Reporter: Martin
Observed: 2026-05-02

## Observation

The V2 player input supports line breaks and scrollability, which is good, but the visible input area only shows one line by default. That makes multiline input feel cramped and a little odd.

## Expected

The player input should likely show at least two lines by default, then continue to support additional internal scrolling for longer input.
