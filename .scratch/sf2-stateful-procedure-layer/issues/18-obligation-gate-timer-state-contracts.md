## What to build

Add or tighten the hard state contracts that prevent procedural pressure from contradicting itself: obligations/debts, resolution-gate satisfaction, and temporal anchor/timer updates.

The 2026-05-10 Forty Thousand playthrough exposed three linked failures. First, lien/debt changes were represented through `credits_delta`, so a 20,000-credit lien increase clamped the player's 120 credits to 0 instead of updating an obligation. Second, gate semantics allowed "lien cleared" to coexist with "wire pending / plate dark" because progress event gate ids were treated like satisfied gate ids. Third, time remained a string label and temporal anchor updates did not apply reliably, so "14 hours" and "10 hours" could coexist.

This ticket creates a genre-neutral state contract for obligations and makes gates/timers explicit enough that pressure cannot move by prose alone.

## Definitions

- **Obligation** - a durable owed thing with owner/claimant, debtor, amount or non-monetary terms, status, due condition, and clearance condition. Debt, tithe balance, blood price, favor owed, ransom, bounty, retainer, and service contract are all obligations.
- **Balance** - numeric amount owed when an obligation is monetary or count-based. Balance is not player wallet currency.
- **Payment event** - structured event that changes an obligation balance or status. It may also change player credits, but those are separate ledgers.
- **Resolution gate event** - structured progress against a thread/procedure gate. Mentioning a gate is not satisfying it.
- **Temporal anchor update** - explicit update/supersession of the canonical time fact, not a fresh text label.

## Acceptance Criteria

- [ ] SF2 can represent a monetary or count-based obligation separately from player credits.
- [ ] `credits_delta` is not used to represent debt/lien/tithe balance changes unless actual player-held currency changes.
- [ ] Obligations support claimant, debtor, balance or terms, status, due condition, clearance condition, and anchors to thread/procedure/document.
- [ ] Narrator/retrieval surfaces active obligations compactly when they are relevant to the current scene.
- [ ] Archivist can create/update/resolve obligations through validated patches, or an existing promise/document/thread shape is deliberately extended to cover the same contract.
- [ ] Resolution-gate handling distinguishes referenced gate ids from satisfied gate ids. Only explicit satisfaction, failure, or waiver marks a gate terminal.
- [ ] Existing progress event fields are migrated or normalized so `gate_ids` does not silently mean `satisfied_gate_ids`.
- [ ] Temporal anchors can be updated or superseded through Archivist patches, including deadline changes.
- [ ] Scene time labels are derived from canonical temporal anchors/countdowns when a live deadline is load-bearing.
- [ ] A Forty Thousand fixture covers 40,000 -> 60,000 as an obligation balance change and rejects "the lien just doubled" math drift if numbers are player-facing.
- [ ] A fixture covers gate cleared vs wire pending and prevents both from being true unless modeled as separate gates.
- [ ] A fixture covers a deadline changing from 14 hours to 10 hours and verifies the old anchor is superseded or updated, not left as parallel truth.

## Agent Brief

**Category:** bug
**Summary:** Represent obligations, gate satisfaction, and deadline updates as hard state instead of prose or wallet deltas.

**Current behavior:** Debt/lien pressure can be represented as player credit deltas, gate progress can be ambiguous between "mentioned" and "satisfied," and deadline changes can survive as parallel text facts instead of superseding canonical time truth.

**Desired behavior:** Obligations should be durable owed-state records distinct from player-held currency. Gate satisfaction/failure/waiver should require explicit terminal transitions. Temporal anchors should update or supersede canonical deadline facts so scene time and timer pressure derive from one active truth.

**Key interfaces:** player credit effects, promise/obligation or new obligation state, thread/procedure resolution gates and progress events, temporal anchors, timer/countdown projection, Archivist patch schemas, retrieval packets, replay fixture assertions.

**Acceptance criteria:**
- [ ] Debt/lien/tithe balances do not mutate player credits unless actual held currency changes.
- [ ] Obligations track claimant, debtor, balance or terms, status, due/clearance conditions, and relevant anchors.
- [ ] Gate references are distinct from satisfied, failed, or waived gate transitions.
- [ ] Deadline changes update/supersede canonical temporal anchors and remove contradictory active countdown truth.
- [ ] Fixtures cover Forty Thousand lien math, gate-cleared-versus-pending semantics, and 14-hour to 10-hour supersession.
- [ ] Existing temporal-anchor, thread-gate, and pressure-event fixtures still pass or are intentionally updated.

**Out of scope:** A full economy system, a general calendar engine, or replacing all promises/decisions unless that is deliberately chosen as the obligation storage contract.
