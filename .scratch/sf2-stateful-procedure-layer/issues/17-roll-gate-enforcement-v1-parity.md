## What to build

Restore V1-level roll discipline in SF2 by making required-roll detection a code-owned contract, not just Narrator prompt guidance.

V1 worked better because the current player message was scanned by `detectRollGate`, a private roll-gate block was injected into the same dynamic state, and `pending_check` lived inside `commit_turn`. The server intercepted `pending_check` before outcome resolution, so the model could not narrate success and persist state in the same final tool call when a check was pending.

SF2 has skill-tag binding and free-text roll gate blocks, but they are advisory unless the Narrator voluntarily chooses `request_roll`. The route accepts `narrate_turn` when no `request_roll` appears. The free-text detector is also narrower than V1 in important places: launch/undock/departure under live constraints, non-trusted NPC information extraction, and movement through already-established danger.

## Definitions

- **Required-roll gate** - deterministic result saying this intent must pause for a roll before outcome narration.
- **Roll gate source** - skill tag, free-text detector, active procedure constraint, NPC disposition/info gate, combat state, or explicit player request.
- **Roll repair** - route-level retry/repair when Narrator emits `narrate_turn` despite a required-roll gate.
- **Roll drought** - advisory signal only. It can nudge cadence, but it is not a substitute for action-specific gates.

## Acceptance Criteria

- [ ] Required-roll gates are computed as structured data before the Narrator call.
- [ ] Narrator context still receives private roll guidance, but route dispatch enforces it.
- [ ] If a required-roll gate exists and Narrator emits `narrate_turn` without `request_roll`, the route repairs, retries, or rejects before visible outcome/state commit.
- [ ] Skill tags remain binding and stricter than free-text heuristics.
- [ ] Free-text gates regain V1 coverage for social manipulation, non-trusted NPC information, technical/system actions, investigation/search, risky movement, physical contests, launch/undock/departure under live constraints, and combat attacks.
- [ ] Favorable NPCs may still require rolls for actionable information unless the state explicitly grants trust or earned disclosure.
- [ ] Pure decisions, trivial movement, downtime, already-earned routes, and no-uncertainty observation remain roll-free.
- [ ] Active procedure constraints can make otherwise-simple movement or departure roll-gated.
- [ ] Roll log records gate source and any repair/override.
- [ ] A Forty Thousand fixture covers "go to cockpit and fly once the seal lifts" under unresolved departure constraints and verifies either a roll prompt or a state-visible reason no roll is required.
- [ ] A social fixture covers asking/pressuring an NPC for actionable information and verifies a roll before the substantive answer.
- [ ] A negative fixture covers a pure acceptance/choice with no roll.

## Historical Agent Brief (superseded by 2026-05-16 verification)

**Category:** bug
**Summary:** Enforce required roll gates at route/runtime level before outcome narration commits.

**Current behavior:** SF2 injects private roll guidance for skill tags and some free-text uncertainty, but the Narrator may still commit `narrate_turn` without a `request_roll`. The server dispatch path does not currently block visible outcome/state commit when a required roll was known before the model call.

**Desired behavior:** Required-roll gates should be structured data computed before Narrator execution. If a gate exists, the route/runtime must ensure a roll prompt occurs before substantive outcome narration and state commit, using repair/retry/rejection where necessary.

**Key interfaces:** roll-gate detector data, skill-tag binding, free-text/action uncertainty classification, active procedure constraints, Narrator route dispatch, roll prompt payloads, roll logs/diagnostics, replay fixture assertions.

**Acceptance criteria:**
- [ ] Required-roll gates are computed as structured data before Narrator calls.
- [ ] Route/runtime enforcement prevents `narrate_turn` from committing a gated outcome without a roll.
- [ ] Skill tags remain binding and stricter than heuristic free-text gates.
- [ ] Free-text coverage includes social pressure, actionable NPC information, technical/search actions, risky movement, constrained launch/departure, and combat attacks.
- [ ] Negative fixtures preserve roll-free pure decisions and no-uncertainty observations.
- [ ] Roll logs expose gate source and repair or override decisions.

**Out of scope:** Changing dice math, adding cosmetic checks, or using roll drought advisories as a substitute for action-specific gates.
