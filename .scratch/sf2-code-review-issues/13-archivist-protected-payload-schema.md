# Type high-risk Archivist payload fields

Status: ready-for-agent
Category: enhancement
**Type:** AFK
**Source:** sf2 code review pass 2, finding A1

## What to build

The Archivist `creates[].payload` is structurally untyped: `additionalProperties: true` with the entity-specific shape described in prose. That keeps the LLM flexible, but it also means high-risk fields drift silently until apply-patch normalization or invariants catch them.

Add structural rails for the most protected fields while preserving the flat-patch model. Start with the fields that have downstream invariants or protected identity semantics: NPC `name`, `pronoun`, `age`, `key_facts`, `supersedes_id`; document `type`, `subject_entity_ids`, attribution fields; temporal anchor `kind`.

## Acceptance criteria

- [ ] Archivist tool schema exposes typed sub-shapes or discriminated documentation for high-risk payload fields
- [ ] `pronoun` accepts only the canonical pronoun anchors or is normalized with a counted fallback
- [ ] `key_facts` has an explicit max of 3 where the schema shape allows it
- [ ] Document `type` is an enum and `subject_entity_ids` is required for document creates
- [ ] Temporal anchor `kind` is an enum
- [ ] Normalization fallback counters are returned in the Archivist response for any coercion still allowed
- [ ] Fixture or schema test proves malformed protected fields are surfaced rather than silently accepted
