# Let coherence findings override bad prior prose

Status: ready-for-agent
Category: bug
**Type:** AFK
**Source:** sf2 code review pass 2, finding D3

## What to build

The Narrator prompt currently says prior prose is authoritative over the scene packet when they conflict. That preserves continuity when state sync lags, but it becomes dangerous when the prior prose was itself flagged by coherence findings such as `npc_fabrication`, `identity_drift`, `pronoun_drift`, or `age_drift`.

Update the Narrator context contract so prior prose remains authoritative only when no coherence finding has already identified that prior prose as wrong. The next turn should repair toward canonical state and the finding, not perpetuate the hallucinated or drifted prose.
