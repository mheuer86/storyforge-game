# Schema-vs-prompt parity test (Archivist as prototype)

Status: ready-for-agent
Category: enhancement
**Type:** AFK
**Source:** sf2 code review pass 2, finding A2

## What to build

The Archivist tool schema and Archivist prompt have drifted on required fields:
- Prompt says "Required every turn: `pacing_classification`, `coherence_findings` (often empty; may contain multiple)" — schema requires only `pacing_classification`
- Prompt says "Every `creates` and `updates` must include `source_quote`" — schema marks `source_quote` optional
- Prompt says "`anchor_miss` is auto-derived ... do NOT emit it manually" — schema enum INCLUDES `anchor_miss`

Build a parity test for ONE role first (Archivist) to design the test shape. The test needs a way to read "the prompt requires X" — likely either:
- A structured comment block at the top of the prompt file (e.g. `// PARITY: required = [pacing_classification, coherence_findings]`) parsed by the test
- A separately-exported `ARCHIVIST_REQUIRED_FIELDS` constant referenced by both prompt-rendering and the test
- An assertion-style block within the prompt that the test extracts

The test compares whatever representation you choose against the tool schema's `required[]` and per-field optionality. Failures point at the specific drift. Demoable: the test fails today on the three known drifts; passes after they're reconciled.

This issue is the prototype. Issue #5 rolls the pattern out to the other four roles.

## Acceptance criteria
