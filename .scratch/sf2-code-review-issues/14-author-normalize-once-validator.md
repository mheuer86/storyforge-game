## What to build

`author/contract.ts` validates Author output twice: `validateChapterRaw(raw, ctx)` and then `validateAuthorSetup(normalizeAuthorSetup(raw), opts)`. The two passes intentionally overlap, and `uniqueErrors` dedupes at the end. This creates two sources of truth for continuation drivers, successor constraints, counts, tension-score rules, and target-turn checks.

Make normalization the single boundary. Normalize the raw tool input once, preserve normalization errors clearly, and run one semantic validator over the canonical `AuthorChapterSetupV2` shape.
