## What to build

The Narrator tool schema documents `set_scene_snapshot.snapshot` with `location_id`, `present_npc_ids`, `time_label`, `established`, and `locked`. The runtime also reads undocumented fields such as `scene_id` and `current_interlocutor_ids`, and accepts some fields either on the parent mechanical effect or nested in `snapshot`.

Make the wire format explicit and narrow. The schema should document every field the reducer consumes, and the reducer should prefer one canonical nesting level while retaining a short compatibility shim for existing fixtures if needed.
