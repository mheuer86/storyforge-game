# Validate Chapter Meaning transition seeds

Status: ready-for-agent
Category: enhancement
**Type:** AFK
**Source:** sf2 code review pass 3, findings G2 and G3

## What to build

Chapter Meaning emits the transition seed consumed by the next Author run. The schema requires the seed fields, but the route mostly trusts the tool input and normalizes missing or malformed values to empty strings, `none`, or `background`.

Add a light validator that sanity-checks the synthesized chapter meaning against current state before returning it to the client. The goal is not to judge prose quality; it is to prevent nonsense transition seeds from creating confusing next-chapter Author validation failures.

## Acceptance criteria

- [ ] Shared `PROCEDURE_NONE = 'none'` constant used by Chapter Meaning and Author procedure budget handling
- [ ] Validator checks `pressure_owner_candidate` against existing NPCs/factions or requires an explicit "new threat/bridge" phrasing
- [ ] Validator checks `do_not_restage` has 2-5 non-empty items when the chapter had procedure residue or resolved milestones
- [ ] Validator checks `procedure_residue.mechanism` and `keep_as` are consistent (`none` does not use `leverage`)
- [ ] Route response includes validation findings or rejects invalid synthesis with a clear error
- [ ] Fixture covers a nonsense pressure owner and shows the validator surfacing it at chapter close
