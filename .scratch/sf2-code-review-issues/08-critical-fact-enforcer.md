# Pick one critical-operational-fact enforcer + remove duplicates

Status: implemented
Category: enhancement
**Type:** HITL
**Source:** sf2 code review pass 1 finding #6, pass 2 finding B3, pass 3 finding E1

## What to build

Fuzzy text matching as policy is currently duplicated across **four** sites:
1. `archivist/route.ts:augmentPatchWithCriticalOperationalFacts` — regex-scans Narrator prose for access-status / diagnostic phrases and synthesizes clue-creates
2. `apply-patch.ts:consumeSupersededClues` — `STALE_FACT_PATTERNS` + `CURRENT_FACT_PATTERNS` with playthrough literals (`0217`, `40200`, `lira voss`)
3. `author/contract.ts:PROCEDURE_GRAVITY_RE` — terms like `countdown|timer|...|beacon|corridor` for the procedure-gravity validator
4. `archivist/prompt.ts` — "Continuity-critical operational facts" section telling the LLM to do the same thing

All four are genre-fragile (overfit to space-opera / hegemony). Fantasy and noir trip none or some. Worse: they contradict — if the Archivist follows the prompt rule, the route regex backstop never fires; if the prompt rule is dropped, all backstops have to be the source of truth.

Decision needed (HITL):

**Option A: Trust the Archivist.**
- Keep the prompt rule (#4)
- Delete the route backstop (#1)
- Delete the apply-patch supersede regex with playthrough literals (#2) — replace with a token-overlap supersede that has no genre-specific words
- Keep `PROCEDURE_GRAVITY_RE` (#3) but make it `procedureGravityForGenre(genreId)`-keyed

**Option B: Code is truth.**
- Strengthen the regex backstops, make them genre-portable
- Drop the prompt rule (#4) since the LLM doesn't need to do work code already does
- Apply-patch literals replaced with a structured "supersedes_id" hint protocol the Archivist can use directly

**Option C: Hybrid.**
- Articulate which kind of facts the Archivist owns vs the regex backstops own
- Document the split, kill the overlap

The current state has all three policies live at once. Pick one.

## Decision

Chosen: **Option C — explicit hybrid**.

Ownership split:
- **Archivist prompt owns capture** of continuity-critical operational facts into clues. The prompt section remains the single source of truth for deciding that a concrete operational result should become a clue.
- **Route backstop is removed.** The Archivist route no longer regex-scans Narrator prose and synthesizes clue creates.
- **Apply-patch owns generic clue supersession only.** It may consume an older stale clue when a new clue clearly sharpens the same fact, but it does not capture new facts from prose and no longer contains playthrough-literal terms.
- **Author procedure gravity remains a validator**, not a clue-capture path, and its trigger vocabulary is split into a shared base plus explicit per-genre lexicons.

Reasoning: the model is best positioned to decide whether a prose fact is durable and player-actionable; code is best positioned to prevent duplicate/stale clue state and to keep procedural openers from becoming mechanism-only scenes.
