# Make NPC merges visible (flag + invariant check)

Status: ready-for-agent
Category: bug
**Type:** AFK
**Source:** sf2 code review pass 2, findings B1 and B4

## What to build

The NPC dedup chain in `apply-patch.ts:534-625` runs five matchers in priority order: `existingById`, `supersedeTarget`, `findMatchingAnonymousNpc`, `findMatchingNpc`, `findMatchingSnapshotPlaceholder`. Three are fuzzy text matchers using `semanticTextOverlaps` (token intersection thresholds).

Two problems:
1. Fuzzy merges fire silently. `driftDedup` only emits on dedup; merges aren't rejections — they're silent collapses with no flag, no instrumentation, no per-merge confidence record.
2. Merges mutate `existingMatch.name`, `existingMatch.retrievalCue`, `existingMatch.lastSeenTurn` directly. `checkNpcIdentity(npc, prior)` is the function that protects names and immutable keyFacts, but it only runs on the create-with-prior path. Merge path bypasses it. A non-anonymous match with a slight name difference silently changes the canonical name.

Make every fuzzy merge:
- Run `checkNpcIdentity` against prior state and reject if a protected field would change
- Emit an `entity_merged` flag (or extend `identity_drift`) carrying `proposedName`, `existingName`, `existingId`, `matcher` (which of the 3 fuzzy matchers fired), and a confidence score
- Surface the flag in `applyArchivistPatch`'s outcomes so the archivist registry shows merge events

## Acceptance criteria

- [ ] `findMatchingAnonymousNpc`, `findMatchingNpc`, `findMatchingSnapshotPlaceholder` (or their consumers in `applyCreate`) call `checkNpcIdentity` before mutating `existingMatch`
- [ ] A protected-field violation on the merge path produces a deferred outcome (not a silent mutation)
- [ ] Each fuzzy match emits a flag with matcher name, confidence, and before/after names
- [ ] The flag flows through `summarizePatchOutcome` and into the archivist route's response
- [ ] Test fixture: an anonymous NPC named "the elder" merges with a `Renn` create — flag emitted, identity check passes
- [ ] Test fixture: an existing `Sova` matches a proposed `Sova Marn` — `checkNpcIdentity` rejects the name change, write deferred
