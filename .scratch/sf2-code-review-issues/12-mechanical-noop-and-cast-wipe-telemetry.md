# Surface mechanical no-ops and cast-wipe recovery

Status: ready-for-agent
Category: bug
**Type:** AFK
**Source:** sf2 code review pass 2, findings C1 and C4

## What to build

`applyMechanicalEffectLocally` returns `void`, and several malformed mechanical effects become silent no-ops. Examples: `inventory_use` for an item the player does not have, `set_location` without `location_id`, and scene changes that omit `present_npc_ids` and therefore clear the cast.

Make these failures visible and recoverable. Silent skips should emit invariant events. A scene change that clears the cast because `present_npc_ids` was omitted should also surface a next-turn recovery or coherence note so the Narrator re-establishes who is present instead of fabricating.

## Acceptance criteria

- [ ] Unknown or unavailable `inventory_use.item` emits `mechanical_effect_no_op`
- [ ] `set_location` without `location_id` emits `mechanical_effect_no_op`
- [ ] Unknown mechanical effect kinds emit `mechanical_effect_no_op`
- [ ] Scene change without `present_npc_ids` emits a recovery/coherence note for the next Narrator turn
- [ ] Fixture covers omitted `present_npc_ids` on scene change and asserts the note is present
- [ ] Existing scene-change fixtures still pass when present NPCs are explicit
