# Make display sentinel repair protocol explicit

Status: ready-for-agent
Category: bug
**Type:** AFK
**Source:** sf2 code review pass 1, finding #7

## What to build

The Narrator route emits `display_sentinel` after the text stream has already reached the client. When visible leaks are repaired, the event currently sends `mode: 'enforce'` plus `repairedProse`. The client knows to replace `proseAccum` with `repairedProse`, but the protocol is implicit and the field name reads like a global enforcement mode.

Make the stream contract explicit: a repaired sentinel event means `repairedProse` supersedes previously streamed text for that turn.

## Acceptance criteria

- [ ] `display_sentinel` includes an explicit `repaired: boolean` field
- [ ] `repairedProse`, when present, is documented as superseding all prior `text` events for the same turn
- [ ] The client uses `repaired === true` rather than `mode === 'enforce'` as the primary branch
- [ ] Backward compatibility for existing logs or fixtures is either preserved or migrated deliberately
- [ ] A replay fixture covers a suggested-action or roll-value leak where streamed prose is replaced by repaired prose
