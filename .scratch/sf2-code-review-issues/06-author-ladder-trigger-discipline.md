## What to build

The Author prompt has rules 21-22 about pressure-ladder discipline that the contract validator does not enforce. Author drifts on these rules in observed playthroughs (memory-flagged: "Storyforge ladder trigger discipline").

Add three sub-checks in `author/contract.ts`'s `validateAuthorSetup`:

**1. Severity at-least-one-hard.** Author tool schema marks `severity` optional and the validator never checks distribution. The prompt says the hard-severity rung is "the chapter's dramatic crystallising beat." Three `'standard'` rungs pass today.
- Reject when `pressure_ladder.filter(s => s.severity === 'hard').length === 0`.

**2. Entity-bound triggers (scene-element regex).** Prompt rule 22: "Never name a specific location, room, doorway, scene object, or scene element by name in the trigger." Triggers naming scene elements die when the chapter moves to a different scene.
- Regex-detect location nouns inside `triggerCondition`: `room|door|hall|chamber|gate|terminal|station|berth|corridor|antechamber|annex|...`
- Fail if matched AND no NPC name from `startingNPCs` appears in the trigger
- Imperfect but better than zero enforcement

**3. target_turns range.** Prompt says 18-25; validator only checks `min < 1 || max < min`. Author can emit `{min: 1, max: 2}` and pass.
- Add bounds: `min >= 12`, `max <= 30`, `max >= min`

Co-locate in one validator extension because they touch the same data structure (`authored.pressureLadder`, `authored.pacingContract`) and the same call site.
